# Verifying The Project

Use the shared verifier for local checks and CI parity:

```bash
python -m venv .venv
. .venv/bin/activate
python -m pip install -e ".[desktop,security,dev]"
python scripts/verify.py
```

For local GUI parity proof on a machine with the desktop extra installed, run:

```bash
python scripts/verify.py --require-real-gui
python scripts/check_real_gui_render.py --require-pyqt6 --timeout-seconds 240 --out-dir artifacts/gui-real
python scripts/check_real_gui_render_artifact.py --artifact-dir artifacts/gui-real
```

That mode keeps the normal verifier scope but passes `--require-pyqt6` into the
real GUI render smoke, so missing PyQt6 or missing live screenshots fail instead
of using the dependency-missing fail-closed branch.

If the live render prints `timed out after ... seconds`, treat the local run as
failed evidence: no GUI screenshot artifact should be promoted from that run.
Use the CI `gui-render` artifact or rerun on a PyQt6-capable desktop where the
bounded command completes and `check_real_gui_render_artifact.py` validates the
captured PNG dimensions, hashes and measured layout/topology evidence.
